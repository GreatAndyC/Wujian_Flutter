# 物见 v1.0.3

这个版本主要是把 Android release 真正往“能持续日用”推进了一步，重点补的是确认链路、图片展示一致性、本地存储稳定性，以及设置页新建配置时那一串 Flutter 生命周期崩溃。

## 下载哪个 APK

| 文件 | 适用设备 | 建议 |
| --- | --- | --- |
| `icheck-v1.0.3-arm64-v8a.apk` | 绝大多数现代 Android 手机 | 推荐下载 |
| `icheck-v1.0.3-armeabi-v7a.apk` | 较老的 32 位 Android 手机 | 老设备备用 |
| `icheck-v1.0.3-x86_64.apk` | Android 模拟器或 x86_64 设备 | 模拟器测试 |

不确定手机架构时，可以用：

```powershell
adb shell getprop ro.product.cpu.abilist
```

包含 `arm64-v8a` 就下载 `icheck-v1.0.3-arm64-v8a.apk`。

## 本版本内容

- 首页待确认队列新增“直接确认”，不必每次都点进详情页再入库。
- 所有主要图片展示区域统一按原始拍摄比例完整显示，减少被裁切后看不全的问题。
- 拍摄页补充点击对焦和镜头切换能力，支持在设备硬件允许时选择广角 / 其他镜头。
- 拍照完成后加入更轻量的右下角缩略图反馈，同时去掉可点击交互，减少拍后卡顿。
- 首页统计卡片改成同一行展示，待确认操作按钮和编辑确认按钮排版更紧凑。
- 视图页新增快速状态编辑，适合批量整理“待确认 / 已装箱”等状态。
- 本地持久化链路补齐：配置字段、物品记录和图片文件都会保存在本地，重新打开 App 后不应因为正常启动而丢失。
- 修复失效图片路径导致的视图页和详情页异常显示问题。
- 修复设置页新建配置时连续触发的 Flutter 依赖断言，包括 `InheritedNotifier` 依赖、文本控制器重建、下拉框强制重建和消息回调时机冲突。
- 修复 iOS 启动阶段 `rootViewController` 弃用告警，并收敛文件保存通道初始化时机。

## 安装

```powershell
adb install -r icheck-v1.0.3-arm64-v8a.apk
```

如果提示签名不一致或版本降级，需要先卸载旧包再安装；卸载会删除本地数据：

```powershell
adb uninstall com.wujian.app.icheck
adb install icheck-v1.0.3-arm64-v8a.apk
```

## 建议验证

1. 安装后先进入“设置”，新建一个配置并点击“测试连接”。
2. 回到首页连续拍几张图，确认加入待确认队列时没有明显卡顿。
3. 直接在首页待确认列表点击“确认”，检查是否正常入库。
4. 进入“视图”页，试一下快速修改状态和批量整理。
5. 完全退出后重新打开 App，确认配置、物品和图片仍然存在。

## 校验

下载后可以用 `SHA256SUMS.txt` 校验 APK 是否完整：

```powershell
Get-FileHash .\icheck-v1.0.3-arm64-v8a.apk -Algorithm SHA256
```

## 注意

- 当前 GitHub Actions release 流程会生成分架构 APK，并自动挂到 tag 对应的 release 页面。
- 如果本机已经安装过更高 `versionCode` 的调试包或旧内测包，安装新包前需要先卸载旧版本。
- 正式上架应用商店前仍需要单独配置 release keystore、版本号策略和商店素材。
